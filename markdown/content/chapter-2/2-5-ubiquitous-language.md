# 2.5. Ubiquitous Language

Es el lenguaje común compartido entre desarrolladores y expertos del negocio que se formula durante el EventStorming para alinear modelos mentales y eliminar ambigüedades. Este lenguaje asegura que tanto la documentación técnica como el código final de SafeStep utilicen exactamente los mismos términos definidos para el dominio empresarial.

| Concepto | Definición |
| --- | --- |
| Alumno | Usuario del sistema con rol académico que busca inscribirse en cursos. |
| Docente | Actor del sistema que tiene acceso a gestionar el silabo y la asistencia en sus cursos asignados. |
| Administrador | Usuario con permisos para gestionar la configuración global de cursos y periodos. |
| Curso | Unidad curricular básica que posee créditos, syllabus y requisitos. |
| Ciclo Académico | Periodo de tiempo específico (ej. 2026-I) durante el cual se desarrollan los cursos. |
| Cupo | Cantidad limitada de vacantes disponibles para un curso en un horario específico. |
| Silabo | Documento oficial que contiene el contenido, objetivos y evaluación de un curso. |
| Matrícula | Proceso formal de inscripción de un alumno en un conjunto de cursos. |
| Pre-requisito | Curso previo que debe estar aprobado antes de inscribir el curso actual. |
| Estado de Pago | Condición financiera del alumno que habilita o bloquea su proceso de matrícula. |
| Módulo de Pagos | Sistema encargado de validar la deuda y procesar transacciones para habilitar la matrícula. |
| Historial Académico | Registro histórico de cursos aprobados y desaprobados que determina la elegibilidad para futuros cursos. |
| Solicitud de Matrícula | Registro temporal generado por el alumno que inicia el proceso antes de la confirmación final. |
| Reporte de Matrícula | Documento o dashboard generado para el administrador con métricas de cupos y alumnos inscritos. |
| Ventanilla única | Canal centralizado de atención donde el alumno puede resolver incidencias técnicas o administrativas. |
