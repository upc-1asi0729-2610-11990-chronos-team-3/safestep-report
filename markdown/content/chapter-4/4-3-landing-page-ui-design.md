# 4.3. Landing Page UI Design

## 4.3.1. Landing Page Wireframe

### 4.3.1.1. Introducción al Wireframe del Landing Page

El wireframe del Landing Page de SafeStep establece la estructura fundamental y el flujo de información antes de aplicar elementos visuales. Este wireframe sirve como plano base que define la posición de todos los elementos y la arquitectura de información que será posteriormente refinada en los mockups.

El diseño del wireframe sigue el principio de "mobile-first" pero prioriza la presentación desktop dado que el Landing Page busca maximizar conversiones tanto en desktop como en mobile. El wireframe está diseñado para ser escalable y mantener su efectividad en ambos formatos de pantalla.

### 4.3.1.4. Regiones del wireframe y su Función

A continuación se detalla cada región del wireframe según su función en el recorrido del usuario:

#### A. Header (Navegación Global)

El header contiene el logo de SafeStep, navegación principal con enlaces a secciones clave, y acciones de conversión (iniciar sesión, comenzar). En desktop es horizontal y visible; en mobile se colapsa en hamburger menu.

**Elementos del header:**
- Logo (enlace al inicio)
- Navegación principal: Características, Simulaciones, Tienda, FAQ
- Botones de acción: Iniciar Sesión, Comenzar Gratis

**Principios aplicados:**
- El logo siempre es visible y clickeable
- Las acciones de mayor frecuencia están a la derecha
- El botón principal ("Comenzar") tiene mayor jerarquía visual

#### B. Hero Section

El hero section es la primera impresión y el punto focal principal. Contiene el mensaje de valor central, llamada a la acción, y prueba social.

**Elementos del hero:**
- Badge de prueba social ("+50,000 usuarios")
- Título principal con gradient impact
- Subtítulo explicativo
- Dos CTAs (botón primario "Comenzar", secundario "Ver cómo funciona")
- Indicators de confianza (avatar de usuarios, rating)

**Principios aplicados:**
- Jerarquía clara: título > subtítulo > CTAs
- Prueba social inmediata
- Dos opciones para diferentes tipos de usuario

#### C. Stats Section

La sección de estadísticas proporciona evidencia cuantitativa del impacto de SafeStep.

**Estadísticas mostradas:**
- Usuarios activos (50K+)
- Módulos de simulación (5)
- Satisfacción (98%)
- Vidas preparadas (12K+)

**Principios aplicada:**
- Números redondos y fáciles de recordar
- Formato visual de grilla para scaneo rápido

#### D. Features Section (Características Principales)

Esta sección presenta las características clave de SafeStep en formato de tarjetas.

**Características mostradas:**
1. Simulaciones interactivas
2. Retroalimentación instantánea
3. Aprendizaje progresivo
4. Gamificación
5. Tienda de productos
6. Acceso multiplataforma

**Principios aplicados:**
- Formato visual uniforme (tarjetas)
- Iconografía consistente
- Brief description supporting cada feature

#### E. How It Works Section (Paso a Paso)

La sección "Cómo funciona" explica el proceso de uso de SafeStep en tres pasos simples.

**Pasos:**
1. Crea tu cuenta
2. Practica simulaciones
3. Obtén certificaciones

**Principios aplicados:**
- Narrativa secuencial (1 > 2 > 3)
- Progresión visual (izquierda a derecha en desktop)
- Conexiones visuales entre pasos

#### F. Simulations Section (Módulos)

La sección de simulaciones muestra los módulos de emergencia disponibles.

**Módulos representados:**
- RCP Básico
- Atragantamiento
- Quemaduras
- Sismos
- Hemorragias
- (+ opción para ver más)

**Principios aplicados:**
- Cards visuales con imagen representativa
- Metadata relevante (duración, rating)
- Call to action hacia "ver todos"

#### G. Gamification Section

Esta sección comunica el sistema de gamificación que mantiene usuarios comprometidos.

**Elementos de gamificación:**
- Sistema de XP
- Insignias
- Rachas
- Rankings

**Principios aplicados:**
- Muestra el sistema de rewards
- Ilustra progreso y logro

#### H. Testimonials Section

Testimonios proporcionan prueba social cualitativa.

**Representación:**
- Avatar de usuario
- Quote del usuario
- Nombre y contexto

**Principios aplicados:**
- Autenticidad percibida
- Formato citas clásico

#### I. FAQ Section (Preguntas Frecuentes)

Las preguntas frecuentes abordan objeciones comunes antes de que el usuario las tenga.

**Estructura:**
- Listado de preguntas/respuestas
- Collapsible para más información

**Principios aplicada:**
- Organización Q&A
- Expandible para no saturar

#### J. Footer

El footer proporciona navegación adicional y enlaces legales.

**Contenido del footer:**
- Links de navegación (Acerca de, Contacto, etc.)
- Redes sociales
- Legal links (Términos, Privacidad)
- Copyright

### 4.3.1.5. Principios de Diseño Aplicados en el Wireframe

El wireframe de SafeStep aplica los siguientes principios de diseño:

#### Jerarquía Visual

- El contenido se organiza por importancia: arriba > abajo
- Tamaño y peso visual reflejan jerarquía
- El "F-pattern" para scaneo de contenido

#### Espaciado Constante

- Uso consistente de gutters y margins
- Grid de 12 columnas en desktop
- Adaptación fluida a breakpoints

#### Diseño Inclusivo

- Tamaño mínimo de elementos táctiles (44x44px)
- Contraste suficiente para legibilidad
- Espacios generosos para interacción

#### Arquitectura de Información

- Navegación clara con etiquetas simples
- Agrupación lógica de contenido relacionado
- Breadcrumbs implícitos en la estructura

## 4.3.2. Landing Page Mock-up

Esta sección presenta y explica los Mock-ups del Landing Page, tanto en su versión para Desktop Web Browser como Mobile Web Browser. En la propuesta y la explicación debe evidenciarse la aplicación de los principios, elementos de diseño, diseño inclusivo y arquitectura de información, así como el Design System establecido para los productos digitales.

### 4.3.2.1. Introducción al Mock-up

El mock-up del Landing Page representa la implementación visual final del wireframe, aplicando el Design System establecido en la sección 4.1. El mock-up utiliza los colores, tipografía, espaciado y componentes definidos en Style Guidelines para crear una experiencia visual coherente con la marca SafeStep.

### 4.3.2.2. Mock-up Desktop

El mock-up desktop del Landing Page aplica los siguientes elementos del Design System:

#### Paleta de Colores Aplicada

**Primary (Azul): #0ea5e9**
- Botones primarios, highlights, iconos principales
- Aplicado en: CTAs, badges activos, hovers

**Safe (Verde): #22c55e**
- Indicadores de éxito, elementos de seguridad
- Aplicado en: estadísticas positivas, ratings

**Accent (Naranja): #f97316**
- Énfasis, alertas moderadas
- Aplicado en: botones secundarios, badges destacados

**Escala de Grises**
- Fondos: White, Gray-50, Gray-100
- Texto: Gray-900 (principal), Gray-600 (secundario)
- Bordes: Gray-200, Gray-300

#### Tipografía Aplicada

**Poppins (Headings)**
- H1: 48-64px Bold - Títulos principales
- H2: 36-48px Bold - Títulos de sección
- H3: 24-28px SemiBold - Subtítulos

**Inter (Body)**
- Body: 16px Regular
- Small: 14px Regular
- Caption: 12px Regular

#### Componentes Visuales

**Tarjetas de características:**
- Fondo: White
- Radio: 16px (rounded-2xl)
- Sombra: shadow-lg
- Padding: 32px
- Hover: translateY(-10px) + shadow-xl

**Botones:**
- Primario: bg-primary, text-white, rounded-full
- Secundario: border-white/30, text-white, rounded-full
- Altura: 48-56px
- Padding: 16px vertical, 32px horizontal

**Inputs:**
- Radio: 12px
- Borde: Gray-300
- Focus ring: primary/20

### 4.3.2.3. Mock-up Mobile

El mock-up mobile mantiene consistencia con desktop mientras adapta la experiencia a pantallas pequeñas:

#### Adaptaciones Mobile

**Diseño responsive:**
- Grid de contenido colapsa a 1 columna
- Navegación colapsada en hamburger menu
- Tarjetas de features en grid 1x6
- Spacing reducido (50-75% de desktop)

**Touch targets:**
- Todos los elementosInteractivos mínimo 44x44px
- Botones altura completa de 48px
- Espaciado entre elementos táctiles 16px

**Optimización visual mobile:**
- Imágenes optimizadas para carga rápida
- Texto legible sin zoom
- Formatos vertical scrolling

### 4.3.2.4. Vistas Detalladas por Sección

#### Hero Section - Vista Detallada

**Desktop:**
- Fondo gradiente: linear-gradient(135deg, #0c4a6e, #0369a1, #0ea5e9)
- Texto: White
- Ilustración flotante a la derecha
- Botones con shadow-lg y pulse animation en CTA principal

**Mobile:**
- Fondo similar
- Texto centrado
- Ilustración hidden o colapsada
- Stacked buttons

#### Features Section - Vista Detallada

**Grid layout:**
- Desktop: 3 columnas
- Tablet: 2 columnas
- Mobile: 1 columna

**Card design:**
- Icono centrado (64x64px)
- Título debajo de icono
- Description debajo de título
- Hover effect con elevación

#### Simulations Section - Vista Detallada

**Card design:**
- Imagen representativa superior (h-48)
- Badge superior ("Popular", "Nuevo")
- Contenido textual inferior
- Footer con metadata

**Categorías de badges:**
- Popular: primary
- Nuevo: accent
- Premium: special gradient

#### Gamification Section - Vista Detallada

**Se características:**
- Fondo gradient azul
- Tarjetas translúcidas
- Iconos grandes prominentes
- Números/stats highlight

### 4.3.2.5. Aplicación del Design System

El mock-up evidencia la aplicación del Design System a través de:

#### Consistencia Visual

- Colores utilizados consistentemente a través de secciones
- Tipografía sigue la jerarquía establecida
- Espaciado uniforme
- Bordes y radios consistentes

#### Diseño Inclusivo

- Contraste suficiente (WCAG AA mínimo)
- Estados de foco visibles
- Tamaños táctiles adecuados
- Texto alternativa para iconos

#### Accesibilidad

- Labels de form claramente asociados
- Campos de input con placeholders útiles
- Botones con texto claro
- Error states visibles

### 4.3.2.7. Estados Interactivos en el Mock-up

Los mockups también muestran los estados interactivos de los componentes principales:

#### Estados de Botón

- **Default**: Color fill, sin sombra
- **Hover**: Sombra incrementada, leve translate up
- **Active**: Escala reducida (0.98)
- **Disabled**: Opacidad reducida

#### Estados de Tarjeta

- **Default**: shadow-md
- **Hover**: shadow-xl, translateY(-10px)
- **Transition**: 300ms ease

#### Estados de Input

- **Default**: Gray-300 border
- **Focus**: Primary ring visible
- **Error**: Red border, mensaje visible
- **Success**: Green border, check icon

### 4.3.2.8. Diseño Responsivo General

El mock-up aplica diseño responsivo a través de:

#### Breakpoints Aplicados

- Mobile: 0-639px
- Tablet: 640-1023px
- Desktop: 1024px+

#### Comportamiento por Breakpoint

- Stacking de grids
- Collapsing de navegación
- Adjustment de espaciado
- Images optimization

#### Imágenes Responsivas

- srcset para diferentes anchos
- Lazy loading en imágenes below fold
- Placeholder durante carga
